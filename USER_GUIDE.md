
# 🦗 Crickets Continuum: Dealer Control Agent (DCA)
### **Official Product Documentation & User Guide (v2.5)**

---

## 📝 **1. Executive Summary: The "Robot-as-a-Copier" Strategy**

The **Crickets DCA** is the industry's first "Translation Layer" between **Autonomous Mobile Robots (AMRs)** and **Legacy Office Equipment ERPs** (e.g., E-automate, Forza).

As dealers pivot from "Selling Boxes" (Copiers) to "Selling Outcomes" (Robotics), they face a data gap: Robots speak complex telemetry, but billing systems only speak "Meter Reads." The DCA closes this gap by making a robot bill exactly like a copier.

---

## 🚀 **2. Core Features**

### 📊 **The Strategic Command Hub**
*   **Fleet Pulse:** Real-time diagnostics, battery health, and node synchronization status.
*   **Profitability Hub:** Automatically converts meters into **Robot Work Units (RWUs)** and calculates Net Margin by subtracting tech labor debt.

### 🧠 **Neural Vault (Consolidated Storage)**
*   **AI Ingestion:** Drop a service manual or a contract into the vault.
*   **DNA Extraction:** The system automatically extracts titles, summaries, and key tags using Gemini Multimodal reasoning.

### 🗺️ **Twin Mode (Twin Sync)**
*   Open the dashboard in two different browser windows. Updates in one (like adding a bot) appear instantly in the other via the **BroadcastChannel API**.

## 💡 **3. Top 5 Demo Scenarios**

### 🎯 **Scenario 1: The "Financial Architect" Audit**
*   **Action:** Click the **Calculator** icon in the "Profitability Hub" or ask Cricket: *"Run a full financial audit."*
*   **Outcome:** Watch the AI analyze the last 30 days of telemetry, normalize it into RWUs, and flag any asset where the "True Cost of Service" (TCOS) exceeds 30% of revenue.
*   **Value:** Proves the app isn't just monitoring health—it's protecting dealer profit.

### 🧬 **Scenario 2: The Neural Ingestion Loop**
*   **Action:** Go to the **Neural Vault**, drag a PDF (any manual or invoice) into the "Ingestion Node."
*   **Outcome:** Watch the terminal logs as the AI "Decodes DNA." The file appears in the grid with a pre-written summary and auto-generated metadata.
*   **Value:** Eliminates manual data entry for contract management and service logs.

### 👯 **Scenario 3: Twin Mode State Sync**
*   **Action:** Open the app in two side-by-side browser windows. Go to **Fleet Simulator** in Window A and click **"Simulate Work Day."**
*   **Outcome:** Watch Window B update its meters and charts instantly without refreshing.
*   **Value:** Demonstrates high-performance multi-window synchronization for control centers.

### 📷 **Scenario 4: Visual Fault Remediation**
*   **Action:** Open the **Cricket Chat** (bottom right), type `/audit` or click the camera. Point the camera at a device (or use the simulated frame).
*   **Outcome:** Click "Audit Frame." The AI identifies the serial number and maps the visual fault to a standard **Copier Service Code (e.g., SC-550)**.
*   **Value:** Shows how field techs can diagnose robots using standard copier logic.

### 📈 **Scenario 5: The ROI Conversion Pitch**
*   **Action:** Ask Cricket: *"I have a customer doing 10,000 deliveries a month. Should I sell them a RaaS contract or a standard lease?"*
*   **Outcome:** The agent uses the **RWU Tiered Billing** logic to calculate the most profitable contract structure for the dealer.
*   **Value:** Turns the dashboard into a sales enablement tool for the dealership's account managers.

## ⚙️ **4. Technical Guardrails**

*   **Uplink:** Direct connection to the **Lexington Production Node**.
*   **Billing:** Default RWU normalization is **1,000 meters = 1 Unit**.
*   **Margin Warning:** System flags any cost-to-revenue ratio over **30%**.
